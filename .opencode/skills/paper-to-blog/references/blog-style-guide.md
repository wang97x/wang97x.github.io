# 论文阅读博客写作风格指南

## 写作原则

### 1. 平衡型风格

兼顾技术深度和可读性：
- ✅ 保留关键公式和技术细节
- ✅ 用通俗语言解释复杂概念
- ✅ 添加实际应用场景
- ❌ 避免过多数学推导
- ❌ 避免过于学术化的表述

### 2. 完整解读

覆盖论文全方面：
1. 背景动机（为什么重要）
2. 核心方法（怎么做）
3. 实验结果（效果如何）
4. 个人见解（价值与局限）

### 3. 图片布局策略

**混合布局**：
- 架构图/流程图 → 紧跟在对应方法介绍后
- 实验结果图 → 放在实验结果章节
- 每张图片配有详细说明

## 格式规范

### 图片说明格式

```markdown
![图片标题](/assets/images/{slug}/xxx.png)
*图 X：说明文字，解释图片展示的核心概念*
```

**要点**：
- 使用斜体 `*文字*`
- 说明图片展示的核心概念
- 解释与上下文的关联

### 数学公式格式

使用 `$$` 包裹 LaTeX 公式：

```markdown
$$G(t_i|a^*)=\log\frac{P(t_i|p,q,a^*,t_{<i})}{P(t_i|p,q,a^{'},t_{<i})}$$
```

### 列表格式

**无序列表**：使用 `-`
```markdown
- 项目 1
- 项目 2
```

**有序列表**：使用数字
```markdown
1. 步骤 1
2. 步骤 2
```

### 引用格式

```markdown
> **关键概念**: 定义或重要说明
```

## 写作技巧

### 1. 标题命名

- 使用问句引发兴趣：`对比解码能否训练更一致的教师？`
- 突出核心发现：`SCOTT 在提升忠实性的同时不牺牲准确率`

### 2. 过渡语句

- `然而，CoT 存在两个严重问题：`
- **发现**:（实验结果后）
- `核心 takeaway`:（总结时）

### 3. 强调重点

**加粗**关键术语和发现：
```markdown
**规模依赖**: 只有在足够大的模型上才能观察到性能提升
**不忠实**: 生成的推理过程与预测答案不一致
```

### 4. 个人见解

体现批判性思维：
- 方法亮点（为什么好）
- 实际应用价值（能用在哪里）
- 局限性（还有什么问题）

## 字数建议

| 部分 | 建议字数 |
|------|----------|
| 一句话总结 | 50-100 字 |
| 背景与动机 | 300-500 字 |
| 核心方法 | 800-1200 字 |
| 实验结果 | 400-600 字 |
| 个人见解 | 300-500 字 |
| **总计** | **2000-3000 字** |

## 常见错误

❌ **过于学术化**
```
我们提出了一个基于对比学习的框架...
```

✅ **通俗易懂**
```
这篇论文提出了 SCOTT，一种让小型模型学会忠实推理的方法...
```

❌ **图片无说明**
```markdown
![pipeline](/assets/images/SCOTT/pipeline.png)
```

✅ **详细说明**
```markdown
![SCOTT 框架总览](/assets/images/SCOTT/pipeline.png)
*图 2：SCOTT 知识蒸馏框架概览。(a) 教师：通过对比解码从大型模型获取一致性推理；(b) 学生：通过反事实推理微调小型模型*
```

❌ **缺少个人见解**
直接结束，没有评价

✅ **包含批判性分析**
```markdown
## 个人见解

### 方法亮点
1. 问题定位精准...

### 局限性
1. 需要访问教师模型的 token 概率...
```
