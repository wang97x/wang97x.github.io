# 技能使用指南 - 如何选择正确的技能

## paper-analyzer vs pdf skill

### 何时使用 paper-analyzer ✅

当用户需要**理解/分析论文内容**时：

**触发关键词**:
- "分析这篇论文"
- "总结这篇论文"
- "理解论文内容"
- "论文阅读"
- "提取论文信息"
- "转换为博客"

**示例输入**:
```
✅ "分析这篇论文：https://arxiv.org/abs/2305.01879"
✅ "总结这篇 PDF 论文的主要内容"
✅ "把这篇论文转换为博客文章"
✅ "论文阅读：https://github.com/user/paper-repo"
✅ "分析这个 PDF 文件，提取关键信息"
```

**输出**:
- 结构化分析结果（标题、作者、摘要、章节）
- 关键贡献提取
- 图片提取和分类
- 可转换为博客文章

---

### 何时使用 pdf skill ✅

当用户需要**操作 PDF 文件本身**时：

**触发关键词**:
- "合并 PDF"
- "分割 PDF"
- "旋转页面"
- "提取表格"
- "添加水印"
- "PDF 转图片"
- "填写表单"

**示例输入**:
```
✅ "合并这两个 PDF 文件"
✅ "旋转 PDF 的第 3 页"
✅ "提取 PDF 中的表格数据"
✅ "给 PDF 添加水印"
✅ "把 PDF 转成图片"
✅ "填写这个 PDF 表单"
```

**输出**:
- 修改后的 PDF 文件
- 提取的表格/文本
- 处理后的图片

---

## 技能优先级规则

### 规则 1：内容理解优先
```
用户提到"论文" + "分析/总结/理解" 
    → paper-analyzer（即使输入是 PDF 文件）
```

### 规则 2：文件操作优先
```
用户提到"PDF" + "合并/分割/旋转/提取"
    → pdf skill
```

### 规则 3：博客转换
```
用户提到"论文" + "博客/文章"
    → paper-analyzer → paper-to-blog
```

---

## 推荐输入格式

### 最佳实践（明确意图）

**论文分析**:
```
"论文阅读：https://arxiv.org/abs/2305.01879 分类：论文阅读，知识蒸馏"
```

**PDF 操作**:
```
"合并这两个 PDF 文件：file1.pdf, file2.pdf"
```

### 避免模糊输入

**不推荐**:
```
❌ "处理这个 PDF"（意图不明确）
❌ "分析这个文件"（未说明是论文还是数据）
```

**推荐**:
```
✅ "分析这篇论文：paper.pdf"
✅ "提取这个 PDF 的表格：data.pdf"
```

## 技能协作流程

### 论文分析完整流程
```
用户输入：分析论文 https://arxiv.org/abs/2305.01879
    ↓
paper-analyzer（下载 + 解析 + 提取）
    ↓
analysis.json + figures/
    ↓
[可选] paper-to-blog（转换为博客）
    ↓
_posts/YYYY-MM-DD-slug-title.md
```

### PDF 操作流程
```
用户输入：合并 PDF file1.pdf file2.pdf
    ↓
pdf skill（合并操作）
    ↓
merged.pdf
```

## 常见问题

### Q: 直接上传 PDF 论文文件，应该用哪个技能？

**A**: 看需求：
- 需要**理解论文内容** → paper-analyzer
- 需要**操作 PDF 文件**（合并/分割/提取表格） → pdf skill

### Q: 两个技能都能提取 PDF 文本，有什么区别？

**A**:
- **paper-analyzer**: 提取后生成结构化分析（标题/作者/摘要/章节），用于理解论文
- **pdf skill**: 提取原始文本，用于进一步处理或转换

### Q: 如何确保 Agent 选择正确的技能？

**A**: 在输入中明确意图：
- 提到"论文/分析/总结" → paper-analyzer
- 提到"PDF/合并/分割" → pdf skill

## 示例对比

### 场景 1：论文理解
**输入**: "帮我分析这篇论文的内容" + PDF 文件
**正确技能**: paper-analyzer ✅
**错误技能**: pdf ❌

### 场景 2：PDF 合并
**输入**: "把这两个 PDF 合并成一个"
**正确技能**: pdf ✅
**错误技能**: paper-analyzer ❌

### 场景 3：论文博客
**输入**: "把这篇论文转成博客文章" + arXiv 链接
**正确技能**: paper-analyzer → paper-to-blog ✅
**错误技能**: pdf ❌

### 场景 4：表格提取
**输入**: "从 PDF 中提取所有表格数据"
**正确技能**: pdf ✅
**错误技能**: paper-analyzer ❌（虽然也能提取，但不是主要用途）

## 总结

| 需求 | 推荐技能 | 原因 |
|------|---------|------|
| 理解论文内容 | paper-analyzer | 结构化分析，提取关键信息 |
| 总结论文 | paper-analyzer | 生成摘要和贡献列表 |
| 论文转博客 | paper-analyzer + paper-to-blog | 完整转换流程 |
| PDF 合并/分割 | pdf | 文件操作专业工具 |
| PDF 转图片 | pdf | 格式转换 |
| 提取表格 | pdf | 表格提取优化 |
| 填写表单 | pdf | 表单处理功能 |

**核心原则**: 
- **理解内容** → paper-analyzer
- **操作文件** → pdf skill
